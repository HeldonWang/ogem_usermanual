Parameter

Syntax

%m_t_aztoncef04 (
inds=<input-dataset>,
whr=<additional-filtered-condition>,
pop_flag=<filtered-population-condition>,
trtgrpn=<numeric-treatment-group-variable>,
pop_mvar=<macro-prefix-label-for-treatment(s)>,
byvar=<by-variable-name>,
var=<analysis-variable>,
alpha=<alpha-value>,
pair=<pairs-sequence>,
debug=<Y|N>
effcount_index=<calculation-index>,
coxlog_out=<output-dataset>,
coxlog_blocklbl=<block-label>,
coxlog_modelkey=<model-key>,
coxlog_alpha=<alpha-value>,
ccoxlog_ensor=<censor-variable>,
coxlog_indicator=<censor-indicator>,
coxlog_ties=<ties-handle-method>,
coxlog_strata=<stratification-variable>,
coxlog_pvalside=<pvalue-sides>,
coxlog_missval=<missing-value-display-text>,
coxlog_digit=<decimal-place-for-ratio-CI>,
coxlog_cidecimal=<CI-decimal-place-increase-from-digit>,
coxlog_hrlbl=<hazard-ratio-label>,
coxlog_cilbl=<confidence-interval-label>,
km_method=<curve-generation-method>,
km_conftype=<confidence-interval-computation-type>,
km_trtgrpnorder=<numeric-treatment-group-variable-order>,
km_cen_num=<CENS-non-event-constant>,
km_step_list=<time-point-list>,
km_step_unit=<step-unit>,
km_outdat_median=<median-dataset-name>,
km_median_label=<median-dataset-display-label>,
km_outdat_timelist=<timelist-dataset-name>,
km_timelist_label=<timelist-dataset-display-label>,
lenlist=<list-of-column-width>,
justlist=<list-of-l|c|r (alignment column: c="center", l="left", r="right")>,
justlist_header=<list-of-l|c|r (alignment header: c="center", l="left", r="right")>,
nolblist=<list-of-Y|N (suppress the display of header text or not)>,
orderlist=<list-of-Y|N (order option used or not)>,
defcol=<description-text-of-rules-for-subtitles-or-columns-name>,
blank_after=<variable-insert-blank-line-after>,
rtf=<rtf-output-Y|N>,
sfx=<suffix>,
);

Parameter

Control input

inds

specifies the input data set.

Item Contents
Default
Value A one or two level dataset name.
Requirement Mandatory
Restriction
Interaction
Eg. adae
adam.adae

whr

specifies filtered condition used in where statement.

Item Contents
Default
Value
Requirement Optional
Restriction
Interaction Filter input dataset inds.
Eg. TRTEMFL="Y"

pop_flag

specifies filtered population, condition used in where statement.

Item Contents
Default
Value
Requirement Optional
Restriction
Interaction Filter input dataset both merge_adsl and inds.
Eg. SAFFL="Y"

trtgrpn

specifies the variable name of numeric value of treatment group.

Item Contents
Default
Value
Requirement Mandatory
Restriction The variable type of trtgrpn must be numeric.
Interaction
Eg. trt01pn

pop_mvar

specifies the macro variable used to class different groups which will be displayed as headers or contents.

Item Contents
Default
Value
Requirement Mandatory
Restriction This should be the same as the gmacro in %m_u_popn.
Interaction Same as pop_mvar in %m_u_desc_stat_grp_mix.
Eg. saspopb

Item Contents

Requirement Optional
Restriction Variable specified by byvar must be character.

Item Contents
Default
Value
Requirement Mandatory
Restriction Variable specified by var must be numeric.

alpha

Item Contents
Default .05
Value
Requirement Optional
Restriction
Interaction

Item Contents
Default
Value
Requirement Optional
Restriction
Interaction

Control effcount analysis

effcount_index

specifies indexes for calcuation including condition, suborder, level and label.

Item Contents
Default
Value list of indexes where each index is quoted by {}, within each index, use leading word: cond=,label=,level=,subord= to sperate.
Requirement Mandatory
Restriction
Interaction
Eg. {cond=CNSR=0 # label=%nrbquote(Total events [a], n (%)) #level=0 subord=1}{cond=cnsr=0 and find(evntdesc,'RECIST progression') label=RECIST progression level=1 subord=2}

coxlog_modelkey

specifies the required statistics to calculate.
Input cox indicates that hazard ratio and confidence interval are required and will be calculated using proportional hazards model.
Input logrank indicates that P-value is required and will be calculated using log-rank test.
Input cox|logrank indicates that hazard ratio, confidence interval and P-value are all required.

Item Contents
Default
Value cox,logrank,`cox logrank`
Requirement Mandatory
Restriction
Interaction
Eg. `cox logrank`

Control reporting

lenlist

specifies the width proportion for each column. The lengths should be separated by "#" and their sum should be less than or equal to 99 per page. If this parameter is not provided, then column length will be calculated in macro automatically. The current algorithm for automatically calculating column widths is not yet mature enough. It is recommended that users manually adjust based on the output situation of RTF.

Item Contents
Default Auto-assigned based on the length of header/content for each column
Value
Requirement Optional
Restriction If specified, all columns must be assigned with value respectively, the number of values separated by "#" must be equal to the number of columns.
Interaction
Eg. 5#2#2#2

justlist

specifies the content alignment for each column, separated by "#". "l" means left-alignment, "c" means center-alignment, "r" means right-alignment.

Item Contents
Default
Value l, c, r for each column.
Requirement Mandatory
Restriction
Interaction
Eg. l#c#c#c

justlist_header

specifies the header alignment for each column, separated by "#". "l" means left-alignment, "c" means center-alignment, "r" means right-alignment.

Item Contents
Default
Value l, c, r for each column.
Requirement Mandatory
Restriction
Interaction
Eg. l#c#c#c

nolblist

specifies whether suppress the display of header name for each column, separated by "#". "Y" means do not display header name.

Item Contents
Default
Value Y, N for each column.
Requirement Mandatory
Restriction
Interaction
Eg. Y#N#N#N

orderlist

specifies whether to apply the ORDER option for each column, separated by "#". "Y" means order variable, which does not repeat the value from one row to the next if the value does not change.

Item Contents
Default
Value Y, N for each column.
Requirement Mandatory
Restriction
Interaction
Eg. Y#N#N#N

defcol

specifies the columns that need to be displayed with each column name separated by a space, or add subtitles between columns (which will be used in proc report column statement). If leave it as null or not to state it, the default values will be used.

Item Contents
Default display all columns
Value
Requirement Optional
Restriction The column name is the variable name in input dataset table which should be character variables and starts from "col".
Interaction
Eg. col1 col2 col3
(col1)("(*ESC*)S={just = center borderbottomwidth=0pt MARGINBOTTOM= 2pt } SUBTITLE (*ESC*)n (N = 100)(*ESC*)S={}" col2 col3)

blank_after

specifies the variable on which "insert blank line" is based.

Item Contents
Default
Value
Requirement Optional
Restriction The variable should exist in input dataset table.
Interaction
Eg. ORDER1

rtf

specifies whether to output rtf report or not.

Item Contents
Default
Value Y, N
Requirement Mandatory
Restriction
Interaction
Eg. N

sfx

specifies the suffix for the output file. If multiple outputs are generated in one code, then sfx is used to distinguish the outputs.

Item Contents
Default
Value
Requirement Mandatory
Restriction
Interaction
Eg. saf
